Migration to monorepo

  • lerna.js
  • Monorepository
  • 40 mb bundle
  • 1 task => n pull requst

The problem
There was a 6 separate projects. First - The portal which is a react app that loads other apps. Being a separate entry point was a requirmens from business side. Another 3 are basic dasboards the 5-th is an our UI Framework and the last one is a proxy server. Everything were in separate repositories and the first painfull thing was having a 5 terminal sessions open just to make it running. The second issue was that task may impact a few module, usualy it UI Framework and one of the app. One task in JIRA - at least 2 pull request in fact. Manual testers had to pay attantion if there are correct versions of each app deployed, of curse there where a havy long pipeline and it just took about 15-20 minutes to deploy it. Why so long? Becouse since UI Framework was a separate project â€“ we had to compile it and inject the buid into outher app. The weight of the final build was almost 40 mb.

The solution
The implemented of the solution took about 50 hour. I moved projects to a monorepo using Lerna.js to make a convinient single script to run and to build the project. It let me acheave the flow like 1 task - 1 pull request, and the ability to connect the UI Framework as a npm package. Despite the dev experience become much better on the project, what was the main goal, also the bundle size and as consiquence the page loading speed decreased in almost 4 times.
